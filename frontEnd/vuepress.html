    # VuePress 搭建个人博客

    # VuePress搭建个人博客，并通过GitHub pages进行部署发布。简易的先搭建了一个雏形，能继续完善的地方还有很多。比如登录，发布新文章，对文章进行评论等。同时还有数据库只是的积累，部署到服务器等。作为TODO 留到下一个阶段来做。先做一个阶段性的总结。

    # 框架的选择

    搭建博客可选择的框架种类很多，最后选择了较为简单易上手的VuePress.

    先附上中文官网网址 https://www.vuepress.cn/

    官网的介绍：VuePress由两部分组成：第一部分是一个极简静态网站生成器，它包含由 Vue 驱动的主题系统和插件 API，另一个部分是为书写技术文档而优化的默认主题，它的诞生初衷是为了支持 Vue 及其子项目的文档需求。事实上，一个 VuePress 网站是一个由 Vue、Vue Router 和 webpack 驱动的单页应用。

    # 全局安装

    npm i vuepress -g

    # 创建项目文件夹

    mkdir blog

    # 初始化项目文件夹

    cd blog npm init -y

    # 添加会用到的文件夹

    添加docs文件夹，docs文件夹中包括.vuepress文件夹（public文件夹（放置静态文件）、config.js文件（配置博客标题，主题，描述，侧边栏等）)和README.md（配置博客首页）文件。docs中创建自己命名的文件夹。存储.md结尾的文章。

    # 配置README.md

    默认主题

    ---
home: true                  
heroImage: /img/background.jpeg    
actionText: 进入我的博客       
actionLink: /frontEnd/  
features:
- title: 前端 - 零碎知识点
  details: 对日常遇到的一些零碎知识点进行总结归纳~
- title: 部署 - 短板
  details: 从前端的发展方向来看，大前端是必然的趋势，必须学习更多的东西吖~
- title: 随笔 - 待定
  details: 不仅分享技术，而且分享生活和心得，共同成长~
footer: Yvonne | 向快乐出发 © 2019.01.11-J      
---

    # 配置config.js

    module.exports = {
    title: '记录与分享&技术与快乐',     // 标题
    description: 'Yvonne‘s blog',  // 描述
    head: [
        ['link', { rel: 'icon', href: '/img/head.jpg' }], // 标签栏里的头像
    ],
    base: '//',   //GitHub仓库名！重要！
    markdown: {
        lineNumbers: false      // 代码块显示行号
    },
    themeConfig: {
        sidebarDepth: 4,
        lastUpdated: 'Last Updated',// 文档更新时间：每个文件git最后提交的时间,
        nav: [                   // 导航栏配置
            { text: '前端', link: '/frontEnd/' },
            { text: '部署', link: '/deploy/' },
            {
                text: '链接',
                items: [           //模拟链接，链到CSDN和GitHub的官网
                    { text: 'csdn', link: 'https://blog.csdn.net/' },
                    { text: 'github', link: 'https://github.com/' }
                ]
            }
        ],
        sidebar: {              // 侧边栏配置
            '/frontEnd/': [        // 组显示，frontEnd文件里的文章一起显示
                '',                 // 即默认显示的README.md文件
                'react',
                'vuepress',
            ],
            '/deploy/': [
                '',
                'test'
            ]
        },
        // sidebarDepth: 2,        // 侧边栏显示2级
    }
}

    # 启动部署项目命令

    在package.json中添加

    
"scripts": {
    "dev": "vuepress dev docs",//启动项目
    "build": "vuepress build docs",//打包项目
    "deploy": "bash deploy.sh"//执行脚本
}

    # 编写脚本 一步推送部署

    在GitHub上创建一个仓库，在blog目录下新建如下deploy.sh文件

    # 自动部署脚本  
set -e
# 构建项目
npm run build
# 进入到构建输出目录
cd docs/.vuepress/dist
git init
git add .
git commit -m 'deploy'
# push到新建仓库的 master 分支

git push -f git@github.com:name/name.git master:gh-pages

    # 执行命令

    npm run deploy

    即可部署成功，输入GitHub中setting中GitHub pages显示的网址就可以在任一台电脑上访问啦~

    # 遇到的问题和注意事项

    所有的静态资源都放在public中。 config文件中base的设置十分重要 部署之后发现首页图片无法显示，本来以为是图片太大了，后来发现可能是网络问题，第二天打开之后正常了，没有深入研究。 md文件中的图片部署之后无法显示，本地可以正常，是路径的问题，使用的是/img/1路径，但是已经在base中设置路径了，还是无法显示。后来使用在线地址解决。
